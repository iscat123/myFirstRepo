linux目录结构：	/sbin	s表示Super User,存放的是系统管理员使用的系统管理程序 \
				/bin	是Binary的缩写，这个目录存放最常用的命令(/usr/bin、/usr/local/bin) \
				/home	存放普通用户的主目录 \
				/root	该目录为系统管理员，超级权限的用户主目录 \
				/lib	动态链接共享库，几乎所有的应用程序都需要的共享库 \
				/lost+found	这个目录一般是空的，系统非法关机后，这里就存放一些文件 \
				/etc	所有的系统管理所需的配置文件和子目录，比如安装mysql数据库 my.conf \
				/usr	用户的应用程序和文件放这个目录，类似与Windows下的program files目录 \
				/boot	存放linux启动所需的核心文件，包括一些链接文件和镜像文件 \
				/proc	【不能动】虚拟目录，系统内存映射，访问这个目录获取系统信息 \
				/srv	【不能动】service缩写，存放服务启动之后需要的数据 \
				/sys	【不能动】该目录安装了2.6内核中新出现的一个文件系统sysfs \
				/tmp	存放一些临时文件 \
				/dev	类似于windows的设备管理器，把所有的硬件用文件的形式储存 \
				/media	linux系统会自动识别一些设备，例如U盘、光驱，这些设备会被挂载到这个目录下 \
				/mnt	系统提供该目录让用户临时挂载别的文件系统，我们可以将外部的存储挂载在/mnt/上，然后进入该目录查看内容 \
				/opt	给主机额外安装然间所摆放的目录。如安装mysql数据库可以放到这个目录下 \
				/usr/local	这个另一个给主机额外安装软件所安装的目录。一般是通过编译源码方式安装的程序 \
				/var	这个目录存放不断扩充的东西，习惯将经常被修改的目录放在这个目录下。包括一些日志文件 \
				/selinux	SELinux是一种安全子系统，它控制程序只能访问特定文件。

rwx权限说明：rwx作用到文件：1.r 代表可以读取文件 \
							2.w 代表可以写入修改文件，但是不可以删除文件 \
							3.x 代表可以执行文件 \
			 rwx作用到目录：1.r 代表可以读取，ls查看目录内容 \
							2.w 代表可以修改，对目录内进行创建+删除+重命名 \
							3.x 代表可以进入，cd \

修改权限指令：说明u:所有者 g:所在组 o:其他组 a:u+g+o \
			  1.chmod u=rwx,g=rx,o=x [文件/目录] \
			  2.chmod u+r,o-x [文件] \
			  3.chmod a-x [文件] 所有用户都去掉执行权限 \
			  4.chmod 755 [文件] \
			  

crond任务调度：crond是后台运行程序，任务调度：是指系统在某个时间执行的特定命令或程序 \
			   crontab进行定时任务设置 \
			   编辑crontab定时任务：crontab -e \
			   查询crontab任务：	crontab -l \
			   删除当期用户所有任务：crontab -r \
			   参数说明(5个占位符*****)：1.第一个*表示，一小时当中的第几分钟 \
										 2.第二个*表示，一天中的第几个小时 \
										 3.第三个*表示，一个月中的第几天 \
										 4.第四个*表示，一年中的第几个月 \
										 5.第五个*表示，一周中的星期几
										 6.*	表示任何时间 第一个*就表示一小时中的每分钟都执行一次\
										 7.,	表示不连续时间 0 2,4,6 * * * 表示每天的2点0分，4点0分，6点0分各执行一次 \
										 8.-	表示连续时间 0 5 * * 1-6 表示周一到周六每天5点0分执行一次 \
										 9.*/n	表示每隔多久执行一次 */n * * * * 表示每隔10分钟执行一次 \
编写crontab任务例子：1.执行crontab -e \
					 2.输入任务到调度文件：*/1 * * * * ls -l /ect/ > /tmp/a.txt 意思是每分钟执行ls命令并> \

磁盘分区：1.IDE硬盘，驱动器标识符为"hdx~"，其中"hd"表明分区所在设备类型(即IDE硬盘)，"x"为盘号(可能是a,b,c,d)，"~"为分区(1,2,3,4) \
		  2.SCSI硬盘，驱动器标识符为"sdx~"，其中"sd"表明分区所在设备类型(即SCSI硬盘)，其他和IDE硬盘一致 \
		  查看所有设备挂载情况：lsblk 或者 lsblk -f \
		  磁盘分区命令：1.fdisk [磁盘] 例：fdisk /dev/sdb \
						2.开始对/sdb分区。 m:显示命令列表；p:显示磁盘分区；n:新增分区；d:删除分区；w:写入并退出 \
						3.说明：开始分区后输入n，新增分区，然后选择p，分区类型为主分区。\
								两次回车默认剩余全部空间。最后输入w写入分区并退出，若不保存退出输入q。\
		  分区格式化：mkfs -t [分区类型] [分区名] 例：mkfs -t ext4 /dev/sdb1 \
		  分区挂载：mount [分区名称] [目录] 例子：mount /dev/sdb1 /newdisk  注意：重启后，挂载会失效，永久挂载通过修改/etc/fstab文件，修改后mount -a 即刻生效\
		  分区卸载：umount [分区名称 或者 目录] 例子：umount /dev/sdb1 或者 umount /newdisk \
		  查询磁盘使用情况：df -h \
		  查询目录的磁盘占用情况：du [选项] [目录] 例子：du -hac --max-depth=1 /opt 说明：-h:带计量单位 -a:含文件 -c:增加汇总值 --max-depth:子目录深度 \
